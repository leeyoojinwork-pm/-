# 내 돈의 흐름 — 배포 가이드

## 1단계: Supabase 세팅

1. [supabase.com](https://supabase.com) 가입 → 새 프로젝트 생성
2. **SQL Editor** 탭 → `index.html` 안 SQL 전체 복사 → Run
3. **Settings → API** 에서
   - `Project URL` 복사
   - `anon public` key 복사

---

## 2단계: GitHub 레포 만들기

```bash
# 터미널에서
cd "프리랜서 회계팀 만들기"
git init
git add index.html
git commit -m "첫 배포"
```

또는 GitHub 웹에서:
1. github.com → **New repository**
2. Repository name: `cashflow` (아무 이름)
3. **Private** 권장 (금융 데이터라서)
4. `index.html` 파일 드래그해서 업로드

---

## 3단계: GitHub Pages 활성화

1. 레포 → **Settings** 탭
2. 왼쪽 메뉴 **Pages**
3. Source: **Deploy from a branch**
4. Branch: `main` / `(root)` → **Save**
5. 잠시 후 `https://[유저명].github.io/cashflow/` 접속 가능

---

## 4단계: 앱에서 Supabase 연결

배포된 URL 접속 → Supabase URL + Anon Key 입력 → 시작

---

## 보안 참고

- GitHub 레포를 **Private**으로 설정해도 Pages URL은 공개돼요
- 완전한 비공개가 필요하면 Netlify Identity나 Cloudflare Access 추가 가능
- Supabase anon key는 읽기/쓰기 가능하지만 URL을 아는 사람만 접근 가능

---

## 업데이트 방법

`index.html` 수정 후 GitHub에 다시 업로드(또는 git push)하면 자동 반영돼요.
